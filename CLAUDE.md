# CLAUDE.md

## What this is
Super Jay RC (display name; internal/codebase name: RCSprint, formerly "RC Dirt Oval") — a browser 3D 1/10-scale dirt-oval RC sprint car game modeled on the **Team Losi 22S Sprint**. Stack: **Babylon.js 7 + Havok (WASM) + Vite + TypeScript**. Driver-stand camera, sim-leaning physics, 15-track career. **Car classes:** the player picks a class on the start screen — **Sport Mod** (the DEFAULT — an IMCA-style open-wheel modified; internal id stays `latemodel` so saves survive the 2026-07-19 rename), **Winged Sprint Car**, or **1:10 RC Off-Road Buggy** — each with its own body, physics baseline, and **independent career** (`src/car/CarClass.ts`). **Tracks:** a start-screen track picker selects the 15-round career **oval**, a self-crossing **figure-8** (at-grade X), or a jump-laden **off-road STADIUM** loop (compact bermed supercross arena with a mix of jump types — tabletop, double, big single, step-up/plateau, whoops) (`src/track/TrackSelect.ts`, `src/track/centerlines.ts`); figure-8/off-road are stand-alone exhibition races. **Day/night is a player choice** (setup-screen toggle; career randomizes it per round) — see Hard rules. **Audio:** procedural **high-revving combustion sprint-car engine** (`src/audio/MotorSound.ts`) — a richer, raspier voice for the *player* car (fundamental + upper harmonics + exhaust-rasp noise) plus a lighter, **stereo-panned, distance-faded** voice for **every AI car**, all under one master gain; mute with **M** / the HUD 🔊 button (persisted), and `setPaused` ramps it to silence while the game is paused.

**Two game modes** (`src/game/Mode.ts`, picked at start after class select): **Career/Sim** (the championship described above — always-advance, points-keyed) and **Arcade** (RC Pro-Am style — a score and a top-3-or-burn-a-continue gate; defaults to the **RC Pro-Am overhead camera** that keeps the player car screen-centered while the track scrolls around it, `src/core/RCProAmCamera.ts`). The earlier on-track box pickups/letters/boost-strip chevrons were **removed**; only the oil/wet **slick patches** remain. **Both modes** start each race with a **ROLLING START** (2026-07-19, replaced the drag-strip light tree + perfect-launch boost): the whole field — player included — rolls off a grid set ~30% of a lap back, **AI-driven in formation**; the moment the FIRST car crosses the start/finish line the green flies ("GREEN GREEN GREEN" banner, flag girl waves) and the player takes control (`launchRace`/`dropGreen` in `main.ts`, `Field.checkLineCross`). **Pause** with **P** / the HUD ⏸ button (freezes sim + race clock + sound); **manual zoom** in every view with the mouse wheel, **+/-** keys, or the on-screen ± buttons on touch.

## Hard rules
- **It ships.** `npm run build` → `dist/` must build clean and run from any static host. Use procedural assets or files under `public/`; no server-side dependencies.
- **Day/night is a PLAYER CHOICE (changed 2026-06-29).** The old "game is always NIGHT" hard rule was **intentionally relaxed at the user's request** — any track can now run day OR night. `main.ts` sets `def.night` as: `?day`/`?night` dev overrides → else EXHIBITION tracks (figure-8 / off-road) follow the setup-screen **Day/Night toggle** (`loadDayNight()`, persisted `rcdirtoval.daynight`) → else **CAREER re-rolls day/night RANDOMLY each round** (`Math.random()<0.5`, re-rolled each playthrough — "change it up"). Night is still the **default** (the toggle defaults to night) and remains the signature look, but daytime on the oval/figure-8 is now allowed and expected. (Historical note: a *2026-pre* accidental daytime regression was reverted then; this 2026-06-29 change is the deliberate replacement — do NOT "restore the forced-night rule" as a fix.)
- **All cars must look picture-perfect** — every car (player and AI) reads as a clean winged sprint car: four corner tires, wing on, body/livery intact, nothing missing/floating/clipping.
- **World scale: ONLY the cars and the track are 1:10.** Every person/prop/building (track marshals, flag girl, drivers'-stand spectators, lawnmower rider, the drivers' stand, the timing booth) is **full real-world size**: 1 unit ≈ 1 ft, a standing adult ≈ **5.7u**, the stand deck ≈ **5u**, a shack/booth ≈ **9u**, a car ≈ **2.5u**. People tower ~2× a car's height. Scale procedural figures at the **feet** (root origin y=0). See the **world-scale** skill — this rule exists because sizing kept getting redone.
- **Verify on screen, don't just describe.** Screenshot (full grid when touching `Car.ts`, `RaycastVehicle.placeWheels`, or spawning) or read sim state before calling a visual/physics change done — see the **screenshot-game** skill.

## Commands
```
npm install
npm run dev      # Vite dev server at http://127.0.0.1:5173
npm run build    # tsc --noEmit (strict) then vite build -> dist/
npm run preview  # serve the production build
```
`npm run build` is the only gate (no test runner/linter). `tsconfig` is strict — `noUnusedLocals/Parameters/noImplicitReturns`, so an unused symbol fails the build. Use `npx tsc --noEmit` for a fast typecheck.

In-game: arrows/WASD drive, R reset, **V (or the upper-left button) cycle camera view (In-Car/Track/Aerial/RC-Pro-Am)**, C quick-toggle aerial, **P pause** (Resume/Restart/Main-Menu overlay), **mouse-wheel / +/- zoom** (all views), G garage/setup, M mute engine sound. **Auto-throttle** (set on the start screen, persisted) = full throttle + steering-only (hides the touch GAS/BRAKE). Gamepad/yoke+pedals take over on actual input. Desktop shows small **V/M keycaps** next to those HUD buttons.

## Project hygiene (rules — these exist because each one cost real time)
- **Anything `src/` imports MUST be committed.** `src/assets/aztec-speedway.png` was statically imported by `OvalTrack.ts` but left untracked; the Pages workflow builds from a **fresh checkout**, so CI failed on the missing import and the live site silently served a stale bundle for weeks while local builds stayed green. After adding any asset: confirm `git status` shows it, and never diagnose "the site looks old" as SW caching before checking the Actions run.
- **Verify the deploy, not just the push.** A green local build says nothing about CI. After pushing to `main`, confirm the GitHub Actions run succeeded before telling anyone the live site is updated.
- **Screenshots go to the session scratchpad, never the repo root.** Root captures accumulated ~90 MB of `run-*.png` debris. The root image/log/txt patterns in `.gitignore` are a safety net, not a license to write there. (`ref-*.jpg` / `run-ref-*.jpg` are the tracked exceptions — real reference photos.)
- **Three docs, three jobs — do not duplicate:** `CLAUDE.md` = architecture + rules, `CHANGELOG.md` = what shipped when, `RESUME.md` = current state + next steps (keep it short; history goes to the changelog).
- **Delete experiments when the winner lands.** Candidate/one-off files (`tools/cand*_body.ts` and friends) get removed once an approach is chosen — they rot into confusing near-duplicates of live code.
- **Two sessions often share this working tree** — never `git add -A`/`.`; stage by name, and expect unrelated modified files from the other session.

## Architecture
- `src/main.ts` — entry point + game-flow state machine (`attract → prerace → racing → finished`); boots engine, loads the career round, builds track/field, runs the fixed-timestep loop (`FIXED = 1/60`, ≤6 catch-up steps). Exposes `window.__field`, `__track`, `__race`, `__marshals` for tests. **Boot/loading progress bar:** the `#loading` splash is a real **app-build progress bar** (`#loadFill`/`#loadLabel`) driven by `setBootProgress(pct, label)` at staged points (engine 10% → physics 20/50% → track/field 85% → ready 100%), then fades out. **Unified setup screen** (`Screens.setup`): ONE start screen sets driver **name + car class + game mode + sound + auto-throttle**, then START — replacing the old class→mode→preRace→name chain. All settings persist (`rcdirtoval.*`) and pre-fill on next launch; changing class/mode saves + reloads with a `sessionStorage["rcdirtoval.autostart"]` flag that then drops straight into the race. **Auto-throttle**: when on, the car runs full throttle and the only input is steering (the touch GAS/BRAKE pedals hide via `input.setAutoThrottle`). **Pause menu** (P / ⏸ → `Screens.pauseMenu`): Resume / Restart / Main Menu — freezes sim + race clock + engine sound; `pausedAccum` keeps the lap clock honest. **Attract reel:** on first open per tab session (`sessionStorage rcdirtoval.seen`) all cars are AI-driven (`Field.attractUpdate`) under a cutting `CinematicCamera`; any click/key sets the flag and reloads into the menu with a fresh grid. Career **always advances** to the next (harder) round — no podium gate.
- **Game modes** (`src/game/Mode.ts`): the player picks **Career/Sim** or **Arcade** at start (after class select). `loadMode`/`saveMode` persist the pick (`localStorage["rcdirtoval.mode"]`, `?mode=career|arcade` override); arcade run-state (`{round, continues, score}`) lives in `localStorage["rcdirtoval.arcade"]`. Start flow: the **unified setup screen** (`Screens.setup`) picks class + mode (+ name/sound/auto-throttle) together; switching class/mode persists + reloads (then autostarts). **Both modes** replace the 3-2-1 text countdown with a drag-strip **light tree** (`Screens.arcadeLightTree`: staging dots → three ambers → GREEN, "GET READY"/"GO!"), firing the green flag at the same ~2.4s moment; a **perfect-launch boost** (gas within ~350ms of green → a brief accel buff, `main.ts`). **Career/Sim is unchanged** — always-advance, championship points. **Arcade** (`src/game/Arcade.ts` = `ArcadeManager`) defaults to the **RC Pro-Am overhead camera** and lays only **oil/wet SLICK patches** on the oval (any car — transient grip loss → slides); the earlier box pickups / boost-strip chevrons / collectible letters were **removed** (`getScore()`/`getLetters()`/`isUpgraded()` survive as no-op stubs). Advancement needs a **top-3** finish; otherwise burn one of ~3 **continues** (out → run resets). `finalize()` in `main.ts` branches by mode. Arcade HUD (`#arcadeHud`, Score / Continues) shows only in arcade mode; `window.__arcade` for tests. See the **arcade-mode** skill.
- **Temporary buff layer on the vehicle** (`src/physics/RaycastVehicle.ts`): `applyBuff(kind:"grip"|"accel"|"top", mult, seconds)`, `grantImmunity(seconds)`, and `buffState()` — ticked/decayed in `update()` and woven into the grip/accel/top-speed math; immunity short-circuits `triggerRollover`. Behavior is identical when no buff is active (so Career/Sim is untouched).
- **The vehicle is custom and kinematic — NOT a Havok body.** `src/physics/RaycastVehicle.ts` integrates its own velocity (slip/friction-circle tire model) + yaw and raycasts the ground for ride height + banking. Havok (`src/physics/PhysicsWorld.ts`) is used **only** for static track collision and wheel rays. Don't make the car a Havok rigid body — that path was abandoned (applyForce desynced velocity from the mesh). Tune via `VehicleConfig`/`DEFAULT_CONFIG` and the slip/grip math.
- Car-to-car contact and wall limits are **positional**, not physics — `src/race/Field.ts` (`resolveContacts`, `wallLimit`), which also owns surface grip, tire wear, dust, and rollovers for the whole field.
- **Rollovers are marshal-gated.** A hard hit (`RaycastVehicle.triggerRollover`) tumbles the car then leaves it **stuck upside down** (`isStuck`, no control/velocity) until a marshal reaches it; `R` (`resetTo`) clears it for the player. `src/race/Marshals.ts` places **6 full-size track marshals**: **2 SEATED in camp chairs at the two infield ENDS** (one chair per end, at x=0; `SIT_DROP` lowers them to read as seated) and **4 STANDING OUTSIDE the track at the four corners** (quarter-turn `cornerS` points offset `W/2 + 4` outward, facing in). Each marshal carries a `seated` flag so it returns to the right posture. Marshals have **varied looks** (`marshalLooks()`/`Look` — different shirt/pants/skin/hair/headwear) so no two read identically (`buildPerson`). Figures are **rigged** — `buildPerson` builds legs on **hip+knee pivots** and arms on **shoulder pivots** (rest pose reads identical to a plain standing figure), exposed via the `personRigs` WeakMap; a marshal **animates a full jog cycle** (legs swing from the hips, knees bend, arms counter-swing) while moving to/from an incident, and returns to rest when standing/seated/working. Every figure has **visible knee joints, shoes (soles at ground), and hands**, parented to the hip/knee/shoulder rig pivots so they swing with the gait — this propagates to all marshals and drivers'-stand spectators; the bespoke figures (flag girl, lawnmower rider) got matching additions (the flag girl's wave-hand rides her flag pivot). World scale is unchanged (~5.7u tall, feet at y≈0). See the **people-anatomy** skill. Static figures (spectators, the streaker's chaser) just leave the rig at rest. Updated each frame from `main.ts` with `field.cars`. They wait until trouble: the **nearest available** marshal (seated or standing) claims a car that is **wrecked (`isStuck`) OR stalled** (speed `< STALL_SPEED` for `> STALL_TIME` while green — tracked in a per-car map), gets up/jogs to it, and **places it back on the racing line** — upright, facing race direction — via `vehicle.resetTo(pos, yaw)` from `track.project()` (the bottom groove), then returns to its post and re-seats/stands. (So marshals reposition onto the track, not just right in place.)
- **A flag girl starts each race.** `src/race/FlagGirl.ts` builds a procedural starter at the start/finish line (`track.sampleAt(track.startFinishS)` — the oval S/F line is **¾ of the way down the front stretch, directly in front of the grandstand**: `centerlines.ts` `startFinishS = L/4`, fixed 2026-07-19 from the old `0.7*(length/2)` units bug that put it at the turn-1 exit; the overhead gantry in `Scenery.ts` now also builds at `sampleAt(startFinishS)` for every shape so line + gantry + flag girl + grid all agree); `flagGirl.greenFlag()` (fired when the rolling-start leader crosses the line, and on the `?demo` start) triggers a big green-flag wave that decays back to idle. Built/updated from `main.ts` like the marshals. **Grid** (`OvalTrack.gridPose`): staggered double-file **~30% of a lap BEHIND the line** so the rolling start has a real AI-paced run to the green; the hero #32 and #11X draw **random grid slots** every race (`Field` ctor).
- **Audio is procedural Web Audio.** `src/audio/MotorSound.ts` synthesizes a **methanol-V8 sprint-car engine on real acoustics**: pitch is the **V8 order-4 firing frequency** (`f = rpm/15`, ~3,000 rpm lopey idle → ~9,000 redline ⇒ 200–600 Hz) driven through the **3-speed gearbox** rev pattern (revs climb per gear, drop on the shift; a true idle takes over below a slow roll); the voice = sawtooth fundamental + harmonics (2nd rasp, 3rd bite, gated **5th "scream"** above ~70% revs) + sub + an **exhaust-rasp** noise band whose **bandpass center tracks RPM**, plus an **idle-lope LFO** (uneven big-cam idle) and a **decel crackle-pop** burst (high-passed noise grains on a sharp throttle lift), through a low-pass that opens with **revs AND load** (keep the cutoff floor high enough to stay audible — a too-low cutoff once silenced it: *re-open the filter*). All under a low master-gain cap. `AudioContext` starts **suspended**; `main.ts` calls `motor.resume()` on the first gesture, then `motor.update(throttle, speed)` each racing step. The **AI field** gets a cheaper tier: `setVoiceCount(n)` → one saw-osc→gain→**stereo panner** voice per car (deterministic per-car detune), `updateVoices(...)` pitches/pans/distance-fades them to the active camera **with a ±4% Doppler bend from listener-relative closing speed** (main.ts finite-differences the camera position for listener velocity). Mute via **M** / `#mute` → `setMuted` (persisted `localStorage["rcdirtoval.muted"]`); `setPaused(b)` ramps the master gain to silence while paused (composes with mute). `window.__audio`; no audio file ships; silent no-op when `AudioContext` is unavailable (headless). See the **engine-sound** skill.
- **Car classes** (`src/car/CarClass.ts`): the player races a **Winged Sprint Car** (`createCar`/`DEFAULT_CONFIG`), a **Dirt Late Model** (`src/car/LateModel.ts` `createLateModel`/`LATE_MODEL_CONFIG` — heavier, planted, full-fendered), or a **1:10 RC Off-Road Buggy** (`src/car/Buggy.ts` `createBuggy`/`BUGGY_CONFIG` — high clearance, long-travel shocks, knobby open wheels, big rear wing; `CockpitCamera` uses a per-class `BUGGY_COCKPIT` eye). `CAR_CLASSES`/`CAR_CLASS_LIST` map each id to its body builder + pristine physics baseline; `loadCarClass`/`saveCarClass` persist the pick (`localStorage["rcdirtoval.class"]`, `?class=sprint|latemodel|buggy` override). `Field` takes a `CarClassDef` and builds every car via `classDef.build`. Careers are **class-keyed** (`Career.ts` `loadCareer(cls)`/`saveCareer(c, cls)` under `rcdirtoval.career.<cls>`, with a one-time migration of the old single-class save into `sprint`). The start screen shows `Screens.classSelect` first; switching class persists + reloads so the field/career rebuild. `SetupPanel` shows the class label. A hidden **`?photo`** mode locks a close rear-3/4 camera onto the player car (shares the post-FX) to actually SEE the bodywork. **Opening splash:** the `#loading` screen shows the real **SUPER JAY #32** racing badge (`public/superjay-32.png`) above the build progress bar; the original shop photo lives in `src/assets/superjay-photo.jpg`.
- **Easter egg:** `src/race/LawnMower.ts` builds a guy on a red riding mower parked on the infield grass (static, built from `main.ts`).
- **Tailgate-party pickup trucks** (`src/race/Pickups.ts`, `buildPickups(scene, shadow, standPosition)` from `main.ts`): procedural trucks backed in behind the grandstand and along the east straight, tailgates down, with 0–2 people sitting on the gate (reuses `buildPerson`/`spectatorLooks`). The drivers' **stand is doubled in length** with a **banner** behind it and a **lit booth window** at night (`Scenery.ts`).
- **AI wing-top numbers + the #42** (`car/Car.ts`): AI cars render their number on the wing-top deck (transparent, ~75%, luminance-contrasting, read from +x); the player keeps the Super Jay #32 logo. One AI slot is always the **red/white #42** (`redOutlineNumber` config → red glyphs with a black outline; set in `Field.ts` PALETTE slot + `Career.ts` `DRIVER_NUMBERS`).
- **Rollovers** now tumble on multiple axes with momentum decay + ground bounce and settle to a natural attitude (`RaycastVehicle.triggerRollover`), still ending **stuck** for marshal/`R` recovery. **Pause** (P / ⏸ → `Screens.pauseMenu`: **Resume / Restart / Quit to Menu** + a live **SETTINGS panel** — sound, auto-throttle, a **graphics-quality selector** that cycles AUTO→MIN…ULTRA→AUTO (manual picks pin the tier via `QualityManager.lockTier`, persisted `rcdirtoval.quality`), and a camera-view cycler — all applied instantly mid-race) and **manual zoom** (all views) live in `main.ts` — see the **camera-system** skill.
- **Post-race replay** (`src/replay/Replay.ts` `RaceRecorder`): `recorder.record()` snapshots every car's pose each physics step during `state==="racing"` into a capped `Float32Array`; a `"replay"` flow drives each car's root from the recorded frames (interpolated, wheels spun from travel) under the cinematic cam with a scrub/play/speed/camera bar (`Screens.replayControls`). A **Watch Replay** button on every results screen enters it; Done returns to results. `state` enum gains `"replay"`; the replay branch suppresses physics/HUD and owns its own camera.
- **Career grids by previous finish** (`Field` `lastRaceOrder` param + `main.ts`): in career mode `career.lastRaceOrder` stores the prior race's finishing order as **identity indices** (from the racer id — `"player"`=0, `"ai{i}"`=i, so a mid-season rename can't misplace anyone), and `Field` maps each identity to a grid slot in that order (winner on pole). `cars[]` stays in identity order so `cars[0]` is always the player. Exhibition/arcade always use the default order.
- **Audio fix** (`MotorSound.resume()`): oscillators start **only after `ctx.resume()` resolves** (starting them while still `suspended` produced silence); player gains bumped ~1.7× for audibility; `enable()` = unmute+resume; the gesture handlers resume on EVERY pointer/key (not `{once}`) and **every menu (setup/pause/results) has a SOUND ON/OFF toggle** that calls `enable()`.
- **Touch controls swapped** (`Input.ts setupTouch`): **steering bar bottom-RIGHT** (stretched `min(60vw,560px)`), **GAS/BRAKE bottom-LEFT**, RESET above the pedals, zoom ± on the right edge; the lower-left dev status box is **hidden on touch** so it doesn't sit under the gas pedal.
- **The car is modeled on a real winged sprinter** (`src/car/Car.ts`): a huge top wing with a **down-swept front scoop** + tall number side boards, **staggered tires** (biggest on the right-rear) on **orange beadlock** wheels, and a detailed tube front end. Tire stagger needs **per-wheel radius** — `WheelDef.radius` feeds `RaycastVehicle.placeWheels` (each tire sits on its own radius); without it all wheels use `cfg.wheelRadius` and big tires sink/float.
- **Track shapes are pluggable** (`src/track/centerlines.ts` `makeCenterline(def)` keyed on `TrackDef.shape`): **oval** (lifted verbatim from the old `OvalTrack.pointAt`), **figure8** (a self-crossing lemniscate with an at-grade X), **offroad** (a winding loop with raised ramp crests → real jumps). `OvalTrack` delegates its centerline to this and **guards oval-only decoration** (infield grass/logo, walls, berm, banners) behind `shape==="oval"`; the 400×400 collidable ground (jump-landing floor), surface, groove, and start/finish are built for all shapes. **`OvalTrack.projectNear(point, sHint, window=60)`** does a windowed projection so figure-8 cars at the coincident X stay on their OWN leg (brute-force `project()` would leg-snap); per-car `lastS` hints are threaded through `RaceManager` (`Racer.lastS`) and `Field`. **Jumps:** `RaycastVehicle` reuses its `airborne`/`vUp`/gravity integrator — it tracks `prevGroundY`/`climbRate` while grounded and converts the ramp climb into a real `vUp` launch at the crest (gated so flat tracks are byte-identical; slope-gravity guarded behind `!airborne`). The **track picker** (`src/track/TrackSelect.ts`: `TrackChoice`, `rcdirtoval.track`, `?track=`, `FIGURE8_DEF`/`OFFROAD_DEF` with off-road `night:false`) is on the setup screen; figure-8/off-road are EXHIBITION (no career/arcade writes; `finalize()` shows a simple result + Watch Replay).
- Tracks are data (`src/track/TrackDef.ts`); `OvalTrack` builds the banked oval (plus a **grassed infield** with the **Aztec Speedway crossed-flags logo sprayed flat onto it** — `src/assets/aztec-speedway.png` (replaced the old Flora Vista `logo.png` 2026-07-05), alpha-blended with faint emissive so it reads at night; the lawnmower easter egg sits on the grass). A **5-tier ladder** (0 Min … 4 Ultra) controls hardware scaling (render resolution via `engine.setHardwareScalingLevel`), MSAA samples, SSAO samples, bloom, and sharpen — each tier's `scaleMul` **multiplies the base** hardware-scaling level so the existing mobile lighter-render path is preserved (mobile just starts/sits lower). `main.ts` constructs `new QualityManager(engine, env.pipeline, env.ssao, coarsePointer ? 1 : 3)` (start tier **High 3** desktop / **Low 1** mobile) and calls `quality.update(engine.getDeltaTime())` **every frame** (not gated by race state). The controller samples `engine.getFps()` every 0.5s with hysteresis (step DOWN if <50 fps sustained ~1s, UP if >58 fps sustained ~3s) and a 2s cooldown. **Manual override:** `lockTier(n)` pins a tier (the FPS controller stands down; `unlock()` returns to auto) — driven by the pause-menu graphics selector, persisted `rcdirtoval.quality`. Every car also gets a **contact-shadow blob** (dark radial ground-plane parented to the chassis) and desktop cars a **rooster-tail clod burst** particle system gated on slip (`Field.makeRoost`). - **Early-career player speed boost** (`main.ts`, right after the field is built): for career levels 1–7 the **player car only** gets `engineForce *= 1 + 0.15*(7-round)/7` (round 0-based) — +15% on level 1 tapering to 0 by level 8. - **Permanent player edge** (`src/car/CarSetup.ts` `applySetup()`): because `applySetup` runs for the **player car only** (Field build + garage re-apply; AI never call it), it's the single place a flat player-only advantage lives, in **every mode and car class**: `engineForce *= 1.05` (~5% faster than any AI car) plus an easier/more-forgiving feel — `steerSpeedFalloff *= 0.85` (responsive at speed), `slipSteer *= 0.8` (less tail-happy), `tireGrip *= 1.06`, `corneringStiffness *= 1.08`. Recomputed from the pristine baseline each call so it never compounds; stacks with the early-career boost above.
- **Off-road jump physics** (`src/physics/RaycastVehicle.ts`): the airborne block converts ramp climb-rate into a launch (cap raised to 16 so big/steep ramps send the car higher than small whoops), pitches the chassis to **follow the jump arc** in the air (`airPitch`), and **settles landings forgivingly** (`landSquat` suspension squat + a small speed scrub on big drops — never a tumble). ## Gotchas (cost real time)
- **Headless sim is slow (~1–5 fps), so render-loop timing misleads.** For deterministic tests, step the sim **synchronously at fixed dt** in `browser_evaluate` and read internal state (`vehicle.heading`, `vehicle.position`) — not `mesh.getDirection()`, which is stale until a render frame. (Adding assets/actors? see the **import-asset** / **add-trackside-actor** skills.)
- **The Playwright MCP browser now screenshots the WebGL canvas CORRECTLY** (2026-07-03; the old "stale/garbled" claim no longer holds) — it's the fastest verify path: navigate, `browser_evaluate` to wait for boot, `browser_take_screenshot`. Headless-Chrome CLI capture breaks when the user's real Chrome is open (profile-lock hang — pass `--user-data-dir=<temp>` + `--no-first-run`).
- **The PWA service worker serves STALE builds during verification** — `vite preview` + a previously-registered SW will hand you the OLD bundle (old titles, old code) no matter how fresh `dist/` is. Before judging a rebuild in a browser, `navigator.serviceWorker.getRegistrations()→unregister()` + `caches.delete(...)`, then reload.
- **Never `DynamicTexture.clone()`** — the clone never reports ready, `scene.executeWhenReady` never fires, and the boot bar sticks at 85% with **zero console errors**. Create a fresh DynamicTexture per use instead.
- **PBR `detailMap` is a trap for canvas noise**: it expects a LINEAR texture (sRGB mid-gray darkens everything ~×0.5) and packs **normal-X in ALPHA** (canvas alpha=opaque = max normal deflection → the whole surface darkens). Reverted — don't re-add without a properly packed texture.
- **A ribbon/fan mesh that "just doesn't render" with all state healthy** (ready/visible/in-frustum, no errors) is usually **winding + backFaceCulling** — test with `material.backFaceCulling = false` before anything else (the infield grass fan needs it).
- **Port 5173 may belong to the OTHER concurrent session's dev server** — its game looks similar but is different code (different `window.__*` globals). Verify against your own build via `npm run preview` (4173) and check the globals match `main.ts`.
- **Backdrops barely show in the racing/driver-stand POV** — that elevated camera follows the car and pitches down into the bowl, so distant horizon silhouettes sit above the frame. They read in the **aerial (C) / attract wide** views; verify backdrop changes there (attract reel via `?round=N`), and keep silhouettes tall/close enough to clear the horizon haze.
- **Backdrop instances must never reach inboard of the outfield** (a wide butte/mesa base sitting at the ring radius will overlap the racing surface and "cut off" the track). `buildBackdrop` clamps each instance's radius with `safeR(r, footprint)` to push it out by half its own width; a footprint is closest to the track along **x** (the z ring is stretched ×`zS`). Trim base/height multipliers rather than relying only on distance.
- **Night moon/stars sit ABOVE the bowl-camera frame** (same reason backdrops barely show). Don't conclude "not rendering" from a `?demo`/attract shot — verify by freezing the scene (`scene.onBeforeRenderObservable.clear()`), aiming a camera at `moon.position`, `scene.render()`, then a Playwright screenshot (`maxZ` must exceed the star-dome radius). See the **night-sky** skill.
- **Don't eyeball figure scale from the racing cameras** — they distort size. Probe a figure's world bounding-box height via `browser_evaluate` (expect a person ≈5.7u, feet y≈0). See the **world-scale** / **verify-goals** skills.
- A **Logitech Flight Yoke + CH Pro Pedals** show up as gamepads; a held button can hijack the keyboard. For deterministic tests, override `navigator.getGamepads = () => []`.
- Babylon `ParticleSystem` defaults to **additive blend** — dust looked like glowing embers until set to `BLENDMODE_STANDARD`.
- **Ground moiré** at grazing angles = over-tiled dirt; use `anisotropicFilteringLevel = 16`, modest tiling, lower `bumpTexture.level`.
- `Matrix.InvertToRef` is not static — use `matrix.invertToRef(out)`.
- Camera `maxZ` must stay above the skybox size or the sky clips to black.
- **Finishing order must rank finished cars by FINISH TIME, not by `progress`.** A finished car's `progress` (`lap*len + s`, `RaceManager`) wraps DOWN at the line (s→0, lap clamped to `totalLaps`), so a progress-only sort ranks the actual winner *behind* cars still on the last lap — the P1 finisher showed as P2+. Sort finished cars first by `finishedAt`, then unfinished by `progress`.
- **Tires/wheels: no sidewall or shoulder mesh may exceed the tread radius.** A torus "bulge" sized past the tread reads as Mickey-Mouse ears. Revolve the tire carcass from a cross-section with `MeshBuilder.CreateLathe` (rounded shoulders, square-ish tread) and set the tire material `backFaceCulling = false` (a revolved surface is single-sided). Hoosier lettering goes on a thin sidewall disc with a punched-out center; the chrome **dished** wheel + center nut shows through.
- **Mobile touch controls must set `touch-action:none` on EACH interactive element** (`Input.ts` `setupTouch` — pad/knob/GAS/BRAKE/RESET), not just the `#touchControls` root: `touch-action` is **not inherited**, and the root is `pointer-events:none` so the touches land on the children. Miss it and a **second finger** (steering while on the gas — happens mid-corner) lets the browser pinch/pan the visual viewport (iOS ignores `user-scalable=no` for zoom), scrolling the `position:fixed` buttons out of view — they "disappear" — and firing `pointercancel` that drops throttle. Also bind `lostpointercapture` so a revoked capture always releases (no stuck/unresponsive button).
- **electron-builder EPERM on the project tree** — building the Windows installer straight into `./release/` under the project dir trips an **EPERM from a filesystem filter** on the tree. The workaround (`npm run build:win` → `scripts/build-win.mjs`): redirect electron-builder's output to a **temp dir**, then copy the finished installer into `./release/`. The `.exe` ships as a **GitHub Release asset**, never committed (`release/` is gitignored).
- **`touch-action:none` + `gesturestart` preventDefault are NOT enough to stop iOS Safari pinch-zoom** when each finger is captured by a control — it still zoomed in real use. The **reliable** block is canceling any **multi-touch** `touchmove`: `document.addEventListener("touchmove", e => { if (e.touches.length > 1) e.preventDefault(); }, {passive:false})` (single-finger scroll stays intact, so the Guide overlay/name input still work). Belt-and-suspenders: **pin `#touchControls` to `window.visualViewport`** (counter-`translate`+`scale(1/scale)` + size to `vv.width/height` on its `resize`/`scroll`) so the controls track the visible area and can never scroll off-screen even if a zoom slips through or the Safari toolbar reflows.

## Distribution
The game ships **two ways** (see `DISTRIBUTION.md` / `README.md`'s Install / Download section):
- **Installable PWA** via `vite-plugin-pwa` — the build emits a `manifest.webmanifest` + service worker that **precaches the bundle incl. the Havok `.wasm`**; icons are generated by `scripts/gen-icons.mjs` from `public/superjay-32.png` (`pwa-192.png`/`pwa-512.png`/`pwa-maskable-512.png`/`apple-touch-icon-180.png`). Installs as an app on **iOS, Android, Windows, and Mac** from the browser. Live: `https://aaronjblair.github.io/RC-Dirt-Oval/`.
- **Windows installer (.exe)** via Electron + electron-builder — `electron/main.cjs` loads the built `dist/index.html`; config in `electron-builder.yml`; built with `npm run build:win` (→ `scripts/build-win.mjs`). Ships as a **GitHub Release asset**, never committed.
- Native iOS/Mac/Android **store** builds are **intentionally not produced** (no Apple hardware/account; iOS can't sideload) — install the PWA instead.
- The **rebuild** skill orchestrates the whole release (docs → build → executables → publish → links).

## Windows / PowerShell
- `npm` is `npm.cmd` — `Start-Process "npm"` fails. Run `npm run dev` as a background process instead.
- PowerShell 5.1 has no `&&`. Chain with `;` or `if ($?) { ... }`.
- **Git is installed but NOT on PATH.** Prepend it before any git command: `$env:Path = "C:\Program Files\Git\cmd;" + $env:Path`. Use `git --no-pager …` for log/diff. The repo's `origin` is the **public** GitHub repo `aaronjblair/RC-Dirt-Oval` (auto-deploys to GitHub Pages on push); commit + push to `main` via the **commit-it** skill.
